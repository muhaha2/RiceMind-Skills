# Fixed DOCX Template for RiceMind Gene Mechanism Reports

Every report generated by this skill must follow this template. Do not omit sections; if a section has no data, state that RiceMind returned no usable records for that section.

## Typography

Apply these Word styles to every generated `.docx`:

| Element | Chinese font | Latin font | Size | Weight |
|---|---|---|---|---|
| Title | SimSun / 宋体 | Times New Roman | 16 pt | Bold |
| Heading 1 | SimSun / 宋体 | Times New Roman | 14 pt | Bold |
| Heading 2 | SimSun / 宋体 | Times New Roman | 12 pt | Bold |
| Body text | SimSun / 宋体 | Times New Roman | 10.5 pt | Regular |
| Table text | SimSun / 宋体 | Times New Roman | 9 pt | Header row bold |
| Figure captions | SimSun / 宋体 | Times New Roman | 9 pt | Regular |

Use Word East Asian font settings for Chinese text and Latin font settings for English, numbers and URLs.

## Title Page

Title:

`{GENE} 的 RiceMind 基因机制证据报告`

Required metadata:

- Generation date.
- RiceMind base URL or statement that the report used a provided RiceMind JSON/MCP payload.
- Target gene query.
- Resolved standard RAP ID or `unresolved` status.
- Retrieval completeness statement.

## 1. 数据来源、检索范围与完整性

Purpose: make the report auditable.

Required content:

- API base URL: `http://lit-evi.hzau.edu.cn/ricemind-api/`.
- Endpoints used.
- Page and limit strategy.
- Total retrieved trait associations.
- Total retrieved sentence evidence records.
- Total unique PMIDs.
- Whether all pages were exhausted.
- Any endpoint error, timeout, reduced retrieval or user-imposed cap.
- If an endpoint fails but other RiceMind evidence is available, state the endpoint error explicitly and continue the report only from the successfully retrieved RiceMind data.

Required table:

`API call summary`: show at most six functionally distinct API calls. Do not list all page-specific URLs. For repeated calls that only differ by `page`, aggregate them by endpoint, query function and page range. Tell the user that compact call summaries and raw returned data are stored in the payload JSON sidecar.

## 2. 基因概貌与外部链接

Data source: `/gene-profile/`.

Required content:

- Input query.
- Official primary ID.
- Standard RAP ID.
- Primary symbol.
- Synonyms.
- CGSNL name when available.
- Cross-database IDs: MSU, Oryzabase, Gramene, Ensembl, protein ID.
- Functional explanation and trait class.
- Chromosome, start, end, strand.
- External platform links.

Required table:

`Gene profile table`: field and value.

Any value under `external_platforms.*`, especially `external_platforms.RAP-DB`, `external_platforms.Ensembl_Plants`, and `external_platforms.Gramene`, must be inserted as an actual Word hyperlink, not plain URL text.

## 3. 全量 GTA 概貌

Data source: `/traits-by-gene/?confidence=All&onto_type=ALL`.

Required content:

- All associated traits returned by RiceMind.
- Grouping by confidence tier.
- Grouping by ontology type.
- Ranking by article support count.
- Earliest publication year when available.

Required figures:

- Confidence-tier distribution.
- Ontology-type distribution.
- Top traits by supporting article count.

Required table:

`Trait landscape summary`: show at most the top 20 trait rows ranked by confidence tier and support count. The complete trait landscape must be written to `normalized_traits.csv`; the DOCX should state this explicitly.

## 4. 证据代码、来源与置信层级统计

Data source: `/traits-by-gene/` and sentence-evidence endpoints.

Required content:

- Evidence-code distribution.
- Source database distribution.
- Tier 1, Tier 2, Tier 3 counts.
- Unique PMID count.
- Statement distinguishing raw evidence code from derived confidence tier.

Required figures:

- Evidence-code distribution.
- Source database distribution.

## 5. 机制证据综述

Data source: `/search-by-trait-and-gene/`, optionally `/sentence-context/`.

Required content:

- Organize the synthesis by mechanism topic, not by raw sentence, raw trait, or confidence-tier list.
- Treat trait labels as indexing terms only; the mechanism prose must be grounded primarily in the full retrieved sentence text saved in `normalized_evidence.csv`.
- Integrate the concrete content of evidence sentences into review-style paragraphs. Do not merely say that a gene is associated with a trait or that sentences co-report gene and trait words.
- Extract sentence-derived mechanism chains such as molecular identity, allele/mutation/loss-of-function language, expression or candidate-locus wording, pathway perturbation, hormone signaling, phenotype endpoints, breeding-use context, stress-growth balance, and developmental context.
- For BPH/brown planthopper resistance reports, organize sentence evidence around resistance loci/QTLs, host-feeding phenotypes, defense signaling, candidate resistance genes, and breeding/introgression workflows. Do not force all BPH evidence into a single cloned-gene mechanism.
- Explain the biological logic connecting the gene, the sentence-derived mechanism chains, and major phenotypes, while making clear that the logic is organized from RiceMind text evidence.
- Mention confidence tier, evidence density and trait labels only as audit context, not as the main content of the mechanism prose.
- Cite PMID sources using bracket style after mechanism statements.
- Keep raw sentences out of this section unless a very short phrase is essential.

Writing rules:

- Use strong wording only for curated/experimental Tier 1 evidence.
- For Tier 2, write "repeated RiceMind literature evidence associates..." or "supports a candidate mechanism..."
- For Tier 3 or NLP-only evidence, write "exploratory RiceMind evidence records..." or "co-reported with..."
- Do not infer mechanisms from general biological knowledge that is absent from the RiceMind payload.
- A good paragraph should read like a literature-review synthesis: for example, "the sentence evidence identifies the gene as X, connects allele or expression changes to Y pathway perturbation, then links that perturbation to Z phenotype or breeding outcome [PMIDs]."

Citation format:

`...[12345678, 23456789]`

## 6. 文献趋势与 PMID 可追溯性

Data source: sentence evidence and article metadata.

Required content:

- Publication-year distribution.
- Unique PMID count.
- Top PMID or article clusters by evidence count when useful.
- Statement that literature frequency reflects publication attention, not necessarily biological importance.

Required figure:

- Publication-year bar chart.

## 7. 品种共现与组学序列信息

Data sources:

- `/varieties-by-gene/`
- `/gene-omics-sequence/`

Required content:

- Variety co-occurrence summary when available.
- Do not include a large `gene-omics-sequence` API returned-summary table in the DOCX body.
- Insert direct, clickable RiceMind `gene-omics-sequence` API URL(s) for the traced RAP ID instead. The full returned sequences should remain in the payload JSON sidecar.
- If useful, state only compact sequence availability in prose; do not paste genomic DNA, cDNA, protein sequence strings, or long transcript tables into the report body.

Do not over-interpret variety co-occurrence as a direct functional mechanism.

## 8. 证据边界与解释限制

Required content:

- Tier 1 / High: curated or experimental support.
- Tier 2 / Medium: repeated NLP/literature or mixed support, discovery candidate.
- Tier 3 / Low: exploratory evidence.
- Sentence co-occurrence alone does not prove causality.
- RiceMind support counts measure published evidence density, not necessarily biological importance.

## Sidecar Files

Every automated report run should save:

- `{gene}_RiceMind_gene_mechanism_report.docx`
- `{gene}_RiceMind_gene_mechanism_report_payload.json`
- `{gene}_RiceMind_gene_mechanism_report_normalized_traits.csv`
- `{gene}_RiceMind_gene_mechanism_report_normalized_evidence.csv`
- `{gene}_RiceMind_gene_mechanism_report_normalized_varieties.csv` when available
- figure PNGs in `{report_stem}_figures/`

The DOCX should not reproduce full normalized trait or evidence tables. It should summarize them and point users to the sidecar files.
