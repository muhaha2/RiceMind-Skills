# Extracted RiceMind API 功能说明

This Markdown file replaces the original user-provided API DOCX in the skill bundle. It preserves the operational API details needed by the RiceMind gene report skill while avoiding duplicate DOCX/Markdown reference files.

Source document date: 2026-06-02.

## Overall Design

RiceMind_Data is a Django REST Framework style data-service app. All APIs are `GET` requests and parameters are passed through URL query strings. Public routes are defined by `RiceMind_Data/urls.py`. Swagger UI is available at `/docs/`.

The default API base URL is controlled by `RICEMIND_API_BASE_URL`; if unset, it is `http://lit-evi.hzau.edu.cn/ricemind-api/`.

Typical request:

`http://lit-evi.hzau.edu.cn/ricemind-api/gene-profile/?gene=sd1`

Data access has three main layers:

- `Entity` / `Sentence`: literature entities and sentence retrieval.
- `UnifiedGene` / `RiceGTA_Knowledge`: gene standardization and gene-trait knowledge graph.
- `RiceGTA_SentenceEvidence` / `FullArticleSentence`: sentence-level evidence and context.
- `RiceOmicsGene` / `RiceOmicsTranscript`: sequence data.

## Additional Route Notes Preserved from the Original API Document

These endpoints are not all required for the core gene mechanism report, but they are retained here because the original API document described them and they may help diagnose retrieval behavior or extend the skill.

| Route | Parameters | Returned shape | Notes |
|---|---|---|---|
| `GET /docs/` | none | HTML Swagger UI | Documentation page generated by `drf_yasg`; not a JSON data endpoint. |
| `GET /all-genes/` | `page`, `limit` | `total_count`, `current_page`, `total_pages`, `genes[]` | Lists distinct gene normalizations from `Entity` where `category='Gene'`, sorted by mention count. The mention count is used for sorting but is not returned. |
| `GET /all-traits/` | `page`, `limit` | `total_count`, `current_page`, `total_pages`, `traits[]` | Lists distinct phenotype/trait normalizations from `Entity` where `category='Phenotype'`, sorted by mention count. |
| `GET /all-varieties/` | `page`, `limit` | `total_count`, `current_page`, `total_pages`, `varieties[]` | Lists distinct variety normalizations from `Entity` where `category='Variety'`, sorted by mention count. |
| `GET /search-by-variety/` | `variety`, `page`, `limit` | `total_count`, `current_page`, sentence records | Retrieves literature sentences by variety entity. Missing `variety` returns HTTP 400. |
| `GET /search-by-trait/` | `trait`, `page`, `limit` | `total_count`, `current_page`, sentence records | Retrieves literature sentences by phenotype/trait entity. Missing `trait` returns HTTP 400. |
| `GET /search-by-variety-and-gene/` | `variety`, `gene`, `page`, `limit` | `total_count`, `current_page`, co-occurrence sentence records | Retrieves sentences where variety and gene entities co-occur. Missing required parameters return HTTP 400. |
| `GET /search-by-variety-and-trait/` | `variety`, `trait`, `page`, `limit` | `total_count`, `current_page`, co-occurrence sentence records | Retrieves sentences where variety and trait entities co-occur. Missing required parameters return HTTP 400. |

General implementation notes from the original document:

- Global list endpoints default to relatively large page sizes, but callers should still pass explicit `page` and `limit`.
- Single-entity sentence searches use entity normalization matching and usually return sentence/article metadata such as `PMID`, `year`, `title`, `journal`, `doi`, `sent_id`, and `text`.
- Co-occurrence endpoints operate through shared sentence-level entity matches rather than causal assertions.
- If a pagination parameter cannot be converted to an integer, Django/Python integer conversion may raise an exception rather than a custom API error.

## Gene Standardization

Gene-centered interfaces call `resolve_gene_to_primary_id(gene_query)`:

1. Exact match to `UnifiedGene.gene_id`.
2. Exact match to `primary_symbol`.
3. Match against `[UPPERCASE_ALIAS]` tokens in `searchable_aliases`.

The function returns `UnifiedGene.gene_id`. Code variables may call this value `rap_id` or `standardized_id`, but the actual return is `gene_id`; if `gene_id` is not a RAP ID and downstream tables use `rap_id`, handle this carefully.

## Common Parameters

| Parameter | Meaning |
|---|---|
| `page` | Page number. Default is usually `1`. |
| `limit` | Page size. Defaults vary: global lists `500`, single gene/variety `50`, single trait `20`, co-occurrence endpoints `10` or `15`. |
| `gene` | Gene query term, standard ID, primary symbol or alias. |
| `trait` | Trait/phenotype query term; usually fuzzy-matched with `trait_description` or phenotype normalization. |
| `variety` | Variety query term. |
| `confidence` | Used by `traits-by-gene`: `High`, `Medium`, `Low`, `All`. |
| `onto_type` | Used by `traits-by-gene`: `ALL`, `GO`, `PO`, `TO`, `CO`, `RTO`. |
| `pmid`, `sent_id`, `window` | Used by `sentence-context`. |
| `rap_id` | Used by `gene-omics-sequence`. |
